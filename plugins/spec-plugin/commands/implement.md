# Implement

## Description

番号指定で実装を開始します。指定された番号のspecフォルダにあるimplementation-plan.mdとtasks.mdに沿って、タスクを順番に実装します。

## Prompt Template

`spec-implement`スキルを使用して、実装を開始してください。

以下のタスクを実行してください：

1. **specフォルダの特定**

   - `.specs/` 配下から `$ARGUMENTS`-* にマッチするフォルダを検索
   - 見つからない場合はエラーを報告

2. **実装計画の読み込み**

   - `implementation-plan.md` を読み込み、変更内容を把握
   - 変更対象ファイル、設計方針、データ構造を確認

3. **タスクリストの確認**

   - `tasks.md` を読み込み、未完了タスク（□）を確認
   - 完了済みタスク（■）はスキップ

4. **タスクの順次実装**

   - 未完了タスクを上から順番に実装
   - 各タスク完了時に tasks.md を更新（□ → ■）
   - implementation-plan.md の内容に沿って実装

5. **完了処理**

   - 全タスク完了後、PLANNINGファイルを削除
   - 実装結果をユーザーに報告

## Notes

- `$ARGUMENTS` には番号（例: 001）が入る
- tasks.md のチェックボックスは □（未完了）と ■（完了）で管理
- 実装中に問題が発生した場合はユーザーに確認する
